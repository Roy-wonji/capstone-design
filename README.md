## 1.개발진  

멋쟁이사자처럼 인하대
- IOS : 정인, 희주, 윤호 
- 디자인: 민주 
- 서버 : 재민

  ### Commit Convention

  ```
  <Type>: <Message>
  ```

  feat: 기능추가  
  feat: 기능  
  fix: 버그 수정  
  refactor: 리팩토링  
  style: 스타일(코드 형식, 세미콜론 추가: 비즈니스 로직에 변경 없음)  
  docs: 문서 (문서 추가, 수정, 삭제)  
  test: 테스트(테스트 코드 추가, 수정, 삭제: 비즈니스 로직에 변경 없음)  
  chore: 기타 변경사항 (빌드 스크립트 수정 등
